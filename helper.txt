Консольные команды

pwd - показывает текущую директорию

cd (change direcrory) - меняет директорию на указанную. Например "cd ~" - сменит директорию на домашнюю папку пользователя. Если в названии директории есть пробелы, нужно использовать кавычки. Чтобы подняться выше (перейти в родительскую директорию) нужно ввести две точки - "cd .." . Если ввести одну точку, мы перейдем в текущую директорию (это иногда нужно для запуска скриптов и других программ). Можно перейти сразу через несколько директорий, нужно использовать "/" для их разделения.

ls (list directory contents) - отображает содержимое директории. Если добавить параметр "-а", то команда покажет расширенное содержимое директории (её скрытые элементы). Также можно смотреть содержимое конкретных папок, не заходя в них, если указать их адрес как параметр - "ls ~" покажет содержимое домашней папки.

Для создания файла используется команда "touch" и имя файла как аргумент (также следует указывать расширение файла)."touch stikhi-to-piska.txt"

Для создания папки (директории) используется команда mkdir (make directory) и имя папки как аргумент. Можно создать структуру директорий с помощью командного флага "-р".

"mkdir -p papka-so-stikhami/stikhi-dlya-piska/luchshee"

Для копирования - "ср" (copy), как аргумент передается файл и место, куда его нужно скопировать.

"cp stikhi-to-piska.txt papka-so-stikhami/"

Можно копировать несколько файлов, главное - не забыть указать конечную директорию.

"mv" (move) - команда для перемещения файлов без копирования, синтаксис тот же, что и у "cp".

"cat" (concatenate and print) - принимает имя файла как аргумент, по выполнению печатает в консоли содержимое файла. Работает только с текстовыми файлами.

"rm" (remove), "rmdir" "rm -r" команды для удаления файлов, папок и папок с содержимым. Команда необратима, использовать с крайней осторожностью.

можно вводить в консоль сразу несколько команд, но нужно разделять их двойными амперсандами "&&". cd ~ && mkdir piska && touch piska/stikh

Можно пользоваться буфером для повторения команд (стрелка вверх или вниз)

Также можно пользоваться автозаполнением (tab). Консоль может дописывать названия директорий и файлов при одинарном нажатии, либо покажет их список, если похожих названий несколько. При двойном нажатии консоль показывает список команд для автозаполнения.

"git init" - инициализация папки как git-репозитория для видимости на git хабе.

"rm -rf .git" - отменить инициализацию (если конкретно, то удалить файл, связывающий папку с гитом, стереть историю изменений и оставить только последнюю версию)

"git status" - показывает состояние папки, её коммиты, ветку, на которой мы находимся.

"git add" - добавляет файл для коммитов. Можно добавить к команде ключ "--all", тогда добавятся все файлы из папки.

"git commit" - непосредственно закомитить изменения в файлах, добавить новую версию. Ключ "-м" позволит добавить сообщение к коммиту (нужно делать это в кавычках), ключ "-а" позволять миновать "git add" для прямого коммита с сохранением изменений в файлах.

"git log" - показывает историю коммитов с сообщениями (если использовался git commit -m), датой и временем.
